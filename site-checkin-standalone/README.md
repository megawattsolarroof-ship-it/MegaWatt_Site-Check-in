# ระบบเช็กอินหน้างาน (Standalone)

ชุดโค้ดพร้อมใช้สำหรับบริษัทใหม่ — แยกออกจากระบบเดิมโดยสมบูรณ์ (ไม่ผูกกับ Firebase หรือ Google Sheet ของบริษัทเดิม)

## มีอะไรในชุดนี้

| ไฟล์ | หน้าที่ |
|---|---|
| `index.html` | หน้าหลัก: เมนู (ลงทะเบียน / เช็กอิน / ตั้งค่า) + ตารางสรุปเช็กอินหน้างานประจำวัน |
| `scan.html` | เช็กอินหน้างาน: สแกนใบหน้า + GPS |
| `register.html` | ลงทะเบียนใบหน้าพนักงาน (บันทึก 3 มุม) — ช่างลงทะเบียนเองได้ |
| `config.html` | ตั้งค่า API URL, App Key, และจุดเช็คอิน GPS — ต้องใส่รหัสก่อนเข้า |
| `js/api-config.js` | ที่วาง URL ของ Google Apps Script |
| `apps-script/Code.gs` | โค้ด backend ทั้งหมด (วางใน Google Apps Script ของ Google Sheet ใหม่) |

> หน้า "ตั้งค่าระบบ" จะถามรหัสทุกครั้งที่เข้า (รหัสอยู่ในตัวแปร `ADMIN_PASS` ใน `config.html`) และ backend ตรวจรหัสซ้ำอีกชั้นตอนบันทึกผ่าน `ADMIN_SECRET`

## ขั้นตอนติดตั้ง (ทำครั้งเดียว ~10 นาที)

### 1) สร้าง backend (Google Sheet + Apps Script)

1. สร้าง Google Sheet เปล่าใหม่ 1 ไฟล์ ด้วยบัญชี Google ของบริษัทใหม่
2. เปิดเมนู **Extensions → Apps Script** → ลบโค้ดเดิมทิ้ง → วางเนื้อหาทั้งไฟล์จาก `apps-script/Code.gs`
3. (แนะนำ) ไปที่ **Project Settings → Script Properties** เพิ่ม 2 ตัว:
   - `APP_SECRET` = รหัสลับอะไรก็ได้ เช่น `newcompany-2569-a8x3k`
     - ถ้าตั้งไว้ → ทุกเครื่องที่ใช้งานต้องกรอกรหัสนี้ในหน้า "ตั้งค่าระบบ" ก่อน (กันคนนอกที่รู้ URL มายิง API เอง)
     - ถ้าเว้นว่าง → ใช้งานได้ทันทีโดยไม่ต้องกรอกรหัส แต่ใครที่รู้ URL ก็เรียกใช้ได้
   - `ADMIN_SECRET` = รหัสเข้าหน้าตั้งค่า — ตั้งเป็นค่าเดียวกับ `ADMIN_PASS` ใน `config.html`
     - backend จะปฏิเสธการแก้ตั้งค่า GPS ที่ไม่แนบรหัสนี้ — ต่อให้ช่างรู้ URL ของ API ก็แก้ค่าไม่ได้
4. กด **Deploy → New deployment → Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
5. คัดลอก **Web app URL** (ลงท้ายด้วย `/exec`) เก็บไว้

> ชีต `Faces` / `Raikhing` / `Engineer` / `Config` จะถูกสร้างให้อัตโนมัติตอนใช้งานครั้งแรก ไม่ต้องสร้างเอง

### 2) ใส่ URL ให้หน้าเว็บ

เปิด `js/api-config.js` แล้ววาง URL ที่ได้จากข้อ 1:

```js
const GAS_API_URL = 'https://script.google.com/macros/s/XXXX/exec';
```

(หรือจะปล่อยว่างไว้ แล้วไปกรอกในหน้า "ตั้งค่าระบบ" ของเว็บทีหลังก็ได้ — แต่ต้องกรอกทุกเครื่อง แนะนำใส่ในไฟล์นี้เลย)

### 3) อัปขึ้น Vercel

โฟลเดอร์นี้เป็น static site ล้วน ๆ ไม่ต้อง build:

```bash
cd <โฟลเดอร์ที่ก๊อปไฟล์ชุดนี้ไปวาง>
vercel --prod
```

หรือลาก-วางโฟลเดอร์ในหน้าเว็บ Vercel ก็ได้ (Framework Preset: **Other**)

> **สำคัญ:** กล้องและ GPS ใช้ได้เฉพาะบน `https://` เท่านั้น — Vercel มี HTTPS ให้อยู่แล้ว จึงใช้ได้เลย

### 4) เริ่มใช้งาน

1. เข้า **ตั้งค่าระบบ** (ต้องกรอกรหัสก่อนเข้า) → กด "ดึงตำแหน่งปัจจุบันของฉัน" ที่หน้างาน → ใส่รัศมี (กม.) → บันทึก
   - ใส่รัศมี `0` = ไม่จำกัดพื้นที่ (เช็กอินจากที่ไหนก็ได้)
   - ถ้าตั้ง `APP_SECRET` ไว้ ให้กรอกช่อง APP KEY ให้ตรงกันด้วย
2. ช่างเข้า **ลงทะเบียนพนักงาน** → ใส่ชื่อ → บันทึกใบหน้า 3 มุมต่อคน
3. พนักงานใช้เมนู **เช็กอินหน้างาน** ได้เลย

## พฤติกรรมการบันทึก

- **เช็กอินหน้างาน** (`Engineer`): เพิ่มแถวใหม่ทุกครั้ง — เช็กอินได้หลายรอบต่อวัน
- ทุกรายการเก็บพิกัด GPS + ลิงก์ Google Maps + หมายเหตุ ลง Google Sheet

## ความแม่นยำของการสแกนใบหน้า

ระบบกันการ "สแกนแล้วขึ้นชื่อคนอื่น" 3 ชั้น:

1. **ต้องจับได้เป็นคนเดิม 3 เฟรมติดกัน** จึงยืนยันตัวตน (เฟรมเบลอ/มุมเอียงเฟรมเดียวจะไม่ถูกตัดสิน)
2. **ต้องชนะคนอันดับ 2 ขาด** — ถ้าคะแนนใกล้กับอีกคนเกินไป จะขึ้นว่า "ใบหน้ากำกวม" ให้เข้าใกล้กล้อง แทนที่จะเดา
3. **ตอนลงทะเบียน** ระบบเตือนถ้าใบหน้าใหม่คล้ายกับคนที่ลงทะเบียนไว้แล้ว และไม่รับภาพที่มัว/มืดเกินไป

ถ้ายังเจอสแกนผิดคน ให้เปิดชีต `Faces` แล้วลบแถวของคนที่มีปัญหาทั้งหมด → ลงทะเบียนใหม่ในที่แสงสว่าง หน้าตรงชัด ๆ ทั้ง 3 มุม

## สิ่งที่ต่างจากระบบเดิม

- **ตัด Firebase Auth ออก** — ระบบเดิมบังคับ login ด้วยบัญชีที่บริษัทเดิมอนุมัติ ชุดนี้เปิดใช้ได้เลย ป้องกันด้วย `APP_SECRET` แทน ถ้าบริษัทใหม่ต้องการระบบ login เต็มรูปแบบ ต้องสร้างโปรเจกต์ Firebase ใหม่แล้วต่อเพิ่มเอง
- ใช้ GAS ตัวเดียวจบ (ระบบเดิมแยก 2 ตัว: ตัวสแกน + ตัว dashboard)
- ตัดเมนู "สแกนเข้า-ออกงาน" ออก — เหลือเช็กอินหน้างานอย่างเดียว
- หน้า "ตั้งค่าระบบ" ต้องกรอกรหัสก่อนเข้า และ backend ตรวจรหัสซ้ำอีกชั้นตอนบันทึกค่า GPS

## หมายเหตุความปลอดภัย

ข้อมูลใบหน้า (face descriptor) ของพนักงานถือเป็นข้อมูลชีวมิติ ควรตั้ง `APP_SECRET` เสมอ เพื่อไม่ให้คนนอกที่เดา URL ได้ดึงฐานข้อมูลใบหน้าหรือลงเวลาแทนคนอื่น
